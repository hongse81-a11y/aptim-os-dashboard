# APTIM OS 주간 수강현황 대시보드 사용 매뉴얼

이 문서는 **'APTIM OS 주간 수강현황 대시보드'**를 관리하고 효율적으로 사용하는 방법을 안내합니다.

## 1. 대시보드 접속 방법

- 배포된 Streamlit Cloud 웹사이트 주소(인터넷 URL)를 통해 접속합니다.
  <https://aptim-os-dashboard-kluefiqcx5tgsjkaygew2x.streamlit.app/>
- 이 URL은 모바일, 태블릿, PC 어디서든 접속 가능하며, 로그인 없이 열람할 수 있습니다.

## 2. 매주 데이터 업데이트 방법 (구글 시트)

대시보드의 데이터는 구글 스프레드시트(**APTIM OS 수강 현황**)와 실시간으로 연동되어 있습니다. 매주 새로운 데이터를 추가하려면 아래의 순서를 따라주세요.

1. **새로운 시트(탭) 생성**: 스프레드시트 하단의 `+` 버튼을 눌러 새 시트를 추가합니다.
2. **시트 이름 변경**: 주차를 알 수 있도록 탭 이름을 변경합니다. (예: `3월 3주`, `3월 4주` 등)
3. **데이터 붙여넣기**: 새로 생성한 시트에 주간 수강현황 결과 등수 표를 붙여넣습니다.
    - **중요**: 데이터의 첫 번째 줄(헤더)에 반드시 `이름` 이라는 글자가 포함되도록 넣어주세요. 시스템이 이 `이름`이라는 글자를 기준으로 데이터 영역표를 자동으로 인식합니다. (윗줄에 빈칸이 있어도 스스로 찾아냅니다!)
    - **필수 컬럼**: `이름`, `완료율`, `평균 등급`, `참여율`, `수강 코스` 항목이 있어야 대시보드에서 정상적으로 추이 분석이 계산됩니다.
4. 웹 대시보드로 돌아가서 브라우저 **새로고침(F5)** 을 누르면, 방금 추가된 최신 시트의 데이터가 실시간으로 그래프와 표에 즉시 반영됩니다.

## 3. 대시보드 주요 기능 사용법

### 1) 기준 주차 선택

- 왼쪽 사이드바의 **[📅 기준 주차 선택]** 드롭다운에서 조회하고 싶은 주차를 선택할 수 있습니다.
- 선택한 주차에 맞춰 요약 수치 및 표 필터링이 즉시 스마트하게 변경됩니다.

### 2) 인원별 심층 분석 (필터기능)

- 왼쪽 사이드바의 **[개별/복수 인원 선택]** 란에서 비교/분석하고 싶은 인원의 이름을 하나 또는 여러 명 선택합니다.
- 선택을 해제하여 빈칸(선택 안 함)으로 두면 **전체 인원의 평균 데이터 추이**를 보여줍니다.
- 선택된 인원들의 평균 수치 및 전주 대비 변화율(%, 점, 개)이 자동으로 계산되어 요약 정보로 표시됩니다.

### 3) 꺾은선 차트 및 막대 차트 (추이 분석)

- 총 4개의 탭(수강 코스, 완료율, 평균 점수, 등급참여율)으로 구성된 시각화 지표를 제공합니다.
- 각 탭을 누르면 선택한 인원들의 시간 경과(주차 별)에 따른 성장 및 변화 추이를 그래프로 직관적으로 확인할 수 있습니다.

### 4) 상세 수강 이력 표 확인

- 메인 화면 하단에 위치한 표는 '선택된 인원 + 기준 주차' 에 해당하는 원본 및 가공 데이터를 표출합니다.
- 표 상단의 **[과거 모든 주차의 이력도 함께 보기]** 체크박스를 체크하면, 숨겨져 있던 과거 이력을 모두 펼쳐 한눈에 표로 비교할 수 있습니다.

## 4. 보고용 PDF 다운로드

대시보드 화면을 그대로 주간 보고서 용도로 스크랩 & 출력하려면 다음의 방법을 사용하세요.

1. 키보드의 `Ctrl + P` (Mac은 `Cmd + P`) 를 누릅니다.
2. 대상 프린터를 **[PDF로 저장]** 으로 변경합니다.
3. 인쇄 설정(기타 설정 부분)에서 **'배경 그래픽 포함'** 옵션을 반드시 체크합니다.
4. 저장을 누르면 화면에 나타난 차트와 예쁜 요약 표가 깔끔한 A4 형태의 PDF 문서로 저장됩니다.

## 5. ⚠️ 시스템 관리를 위한 주의 사항

- **구글 시트 헤더(제목) 임의 변경 주의**: `완료율`, `평균 등급`, `참여율`, `수강 코스` 와 같이 대시보드에서 기준으로 삼고 있는 핵심 열(Column)의 이름이 시트에서 임의로 변경되면 차트가 그려지지 않을 수 있습니다.
- **시트 공유 권한 유지**: 시스템이 구글 시트를 지속적으로 찾을 수 있도록 파일의 공유(편집자 또는 뷰어 권한)에 생성된 '서비스 계정 (ai-news-editor...)' 이 영구적으로 유지되어야 합니다. 삭제하지 않도록 조심해주세요.
